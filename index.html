-- Methods vs Computed Properties

  Um objeto vue pode ter uma sessao computed que tem um funcionamento similar a sessao methods com alguns detalhes
  interessantes ... Quando o dom sofre um alteracao os as funcoes que estao contidas na sessao methods sao executadas
  obrigatoriamente ja que nela o view nao qualquer controle sobre o conteudo da mesmas.
  Ja na sessao computed o view faz uma especie de cache das funcoes e tem conhecimento do conteudo das mesmas 
  senso capaz executar so aquilo que e necessario para a atualizacao necessaria do dom .... 
  E muito util para funcoes complexas que nao precisam serem atualizadas a todo o momento.
  Um outro detalhe e que a funcao contida na sessao computed nao eh chamada como uma funcao mas sim como uma variavel.
  
  Observacao importante :

  As computed properties devem ser utilizadas para resultados sincronos ou praticamente sincronos

-- Sessao watch

A sessao watch do objeto VUE serve como o proprio nome sugere para chamar uma funcao que roda
de maneira assincrona toda a vez que uma "watched property" da sessao data é alterada.
Segue a sintaxe basica:

watch: {
    propriedadedodata: function(newvalue,oldvalue){ // o value e passado de maneira automatica pelo vue
            var vm =this; // como a chamada e feita como um call back e preciso colocar
            a referencia do objeto vue original se voce quiser alterar algo nele
            vm.propriedadedodata=0;
    }
}
